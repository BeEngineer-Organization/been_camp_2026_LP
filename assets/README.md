# Assets フォルダ

このフォルダには、Webサイトで使用する画像やロゴを保存します。

## 📁 フォルダ構成

```
assets/
├── fav/        # ロゴ・ファビコン・キャラクター（SVG）
├── photos/     # 合宿の写真（詳細は photos/README.md）
│   ├── hero/
│   │   └── hero-01.jpg 〜 hero-14.jpg   # ヒーロー周囲コラージュ
│   └── voice/
│       └── voice-01.jpg 〜 voice-06.jpg # 昨年度の様子ギャラリー
```

## 🖼️ 推奨画像サイズ

### ヒーロー・ギャラリー写真
- **推奨サイズ**: 800×600px 前後
- **フォーマット**: JPG (軽量化推奨)
- **用途**: トップ周囲コラージュ・昨年度の様子

### ブログ写真
- **推奨サイズ**: 800×600px 程度
- **フォーマット**: JPG (軽量化推奨)
- **用途**: 合宿の様子を伝える写真

### ロゴ
- **推奨サイズ**: 500×500px 以上（正方形または横長）
- **フォーマット**: PNG（透過背景推奨）
- **用途**: サイドバーやヘッダー

## 📝 画像の追加方法

1. 適切なフォルダに画像を保存
2. HTMLファイルで画像のパスを指定

```html
<!-- 例: ヒーロー -->
<img src="assets/photos/hero/hero-01.jpg" alt="">

<!-- 例: 昨年度の様子ギャラリー -->
<img src="assets/photos/voice/voice-01.jpg" alt="昨年度合宿の様子">
```

## 🎨 画像最適化のヒント

- **圧縮**: [TinyPNG](https://tinypng.com/) などで画像を圧縮
- **サイズ調整**: 必要以上に大きい画像は避ける
- **WebP形式**: 対応ブラウザ向けにWebP形式も検討
- **遅延読み込み**: 大量の画像がある場合は `loading="lazy"` を追加

```html
<img src="assets/photos/photo.jpg" alt="説明" loading="lazy">
```

## 📌 注意事項

- 画像の著作権に注意してください
- 人物が写っている場合は、掲載許可を取得してください
- ファイル名は英数字とハイフンのみ使用（日本語は避ける）
  - ✅ 良い例: `day1-study.jpg`
  - ❌ 悪い例: `1日目の勉強.jpg`
